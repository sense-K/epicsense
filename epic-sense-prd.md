# Epic Sense — 에픽세븐 게임 도구 플랫폼 PRD

## 1. 서비스 개요

### 1.1 서비스명
- **서비스명**: Epic Sense
- **도메인**: 미정 (추후 결정)
- **슬로건**: 에픽세븐 유저를 위한 게임 도구 플랫폼

### 1.2 서비스 목적
에픽세븐(Epic Seven) 유저가 실시간 아레나 드래프트, 장비 세팅, 장비 강화 판단을 쉽고 빠르게 할 수 있도록 돕는 팬 사이트. SEO 최적화를 통한 자연 유입을 주 트래픽 전략으로 삼는다.

### 1.3 수익 모델
- Google AdSense 광고

### 1.4 타깃 유저
- 에픽세븐 실시간 아레나(실레나) 참여 유저
- 장비 세팅 및 강화에 어려움을 느끼는 뉴비~중급 유저
- 글로벌 유저 포함 (한국어 우선, 영어 확장 고려)

## 2. 기술 스택

### 2.1 프론트엔드
- **Framework**: Next.js 14 (App Router)
- **Styling**: Tailwind CSS
- **Language**: TypeScript
- **배포**: Cloudflare Pages

### 2.2 백엔드/데이터베이스
- **Database**: Supabase (PostgreSQL)
- **Auth**: Supabase Auth
- **API Proxy**: Cloudflare Workers (e7api.onstove.com 프록시)

### 2.3 외부 API
- **에픽세븐 공식 데이터**: `https://e7api.onstove.com/gameApi/`
  - POST 방식, `Caller-Id: WEB_STOVE_EPIC7` 헤더 필요
  - CORS 이슈로 Cloudflare Worker를 통한 서버사이드 프록시 필수
  - 주요 엔드포인트:
    - `getSeasonList?lang=ko` — 시즌 목록
    - `getPopularHero?season_code=&grade_code=&lang=ko` — 인기 영웅 목록
    - `getHeroAnalysis?hero_code=&season_code=&grade_code=&lang=ko` — 영웅 분석
    - `getHeroMasterAnalysis?hero_code=&season_code=&lang=ko` — 영웅 심층 분석 (장비 세트, 법 배로, 연운 상성)
    - `epic7_hero.json` — 전체 영웅 기본 데이터

### 5.2 실레나 서포터 (/rta)

#### 목적
실시간 아레나(RTA) 드래프트 중 최적의 픽을 추천해주는 핵심 기능.

#### 데이터 소스
- 공식 API 실시간 연동 (`getHeroMasterAnalysis`, `getPopularHero`)
- 시즌/등급별 데이터 제공

#### 기능 상세

**설정 영역 (상단)**
- 영웅 풀 선택: 전체 영웅 / 보유 영웅 (로그인 시 보유 영웅 기능 활성화)
- 프리밴 수 선택: 1명 / 2명 / 3명 / 4명
- 픽 순서 선택: 선픽 / 후픽
- 시즌 선택: API에서 가져온 시즌 목록 드롭다운
- 등급 선택: 마스터 / 챔피언 / 그랜드마스터 등
- 전체 초기화 버튼

**픽 순서 로직**
```
선픽: 내(1) → 상대(2) → 상대(3) → 내(4) → 내(5) → 상대(6) → 상대(7) → 내(8) → 내(9) → 상대(10)
후픽: 상대(1) → 내(2) → 내(3) → 상대(4) → 상대(5) → 내(6) → 내(7) → 상대(8) → 상대(9) → 내(10)
```

**현재 턴 표시**
- 상단 인디케이터 바: "N픽: 내 영웅을 선택하세요" or "N픽: 상대 영웅을 입력하세요"
- 현재 활성 슬롯에 pulse 애니메이션 (파란색: 내 픽, 빨간색: 상대 픽)

**내 픽 / 상대 픽 슬롯 (좌우 패널)**
- 각 5개 슬롯
- 슬롯에 픽 번호 표시 (1픽, 4픽 등 실제 드래프트 순서)
- 선택된 영웅: 이름 + 픽률 표시
- 현재 선택 대기 슬롯: pulse 애니메이션
- 미래 슬롯: 빈 상태

**영웅 선택 그리드 (중앙)**
- 필터: 속성(불/물/땅/빛/어둠), 직업(전사/기사/마법사/도적/레인저/영혼)
- 영웅 아이콘 그리드 (데스크탑: 10열, 모바일: 6열 2줄)
- 이미 선택된/밴된 영웅: 흐리게 표시, 클릭 불가
- 영웅 클릭 시 현재 활성 슬롯에 자동 추가

**추천 영역 (중앙 하단)**
- 좌측: "함께 쓰면 좋은 영웅" — 내가 선택한 영웅 기준 시너지 top5
  - 영웅 이미지/아이콘 + 이름 + 함께 사용률(%)
  - 클릭 시 바로 내 픽에 추가
- 우측: "카운터 추천" — 상대가 선택한 영웅 기준 카운터 top5
  - 영웅 이미지/아이콘 + 이름 + 상대 승률(%)
  - 클릭 시 바로 내 픽에 추가
- 데이터 출처 표시: "스마일게이트 공식 전적 · {등급} · {시즌}"

**모바일 레이아웃**
- 설정 영역: 가로 스크롤 가능한 chip 형태
- 내 픽/상대 픽: 상단 2열 그리드, 슬롯 5개씩 가로 배치 (스크롤 없이 한 화면)
- 영웅 그리드: 6열 2줄 (최소화)
- 추천: 2열 카드

### 5.3 영웅 도감 (/heroes)

#### 목적
에픽세븐 전체 영웅 정보를 탐색하고 상세 정보를 확인하는 페이지.

#### 목록 페이지 (/heroes)
- 검색창: 영웅 이름 검색
- 필터: 속성 / 직업 / 등급(5성/4성/3성)
- 영웅 카드 그리드: 영웅 이미지 + 이름 + 속성 + 직업
- 카드 클릭 시 상세 페이지 이동

#### 상세 페이지 (/heroes/[hero_code])

**상단 영웅 프로필**
- 영웅 일러스트 배너 (공식 사이트 이미지)
- 영웅 프로필 이미지 (원형)
- 영웅명, 속성, 직업, 별자리 아이콘
- 픽률, 승률, 밴률 (API 실시간)

**주요 착용 세트**
- 공식 API `getHeroMasterAnalysis` 데이터 활용
- 세트 조합 아이콘 + 사용 비율(%) + 승률
- 상위 5개 세트 표시

**추천 세팅 스탯 구간**
- 공식 API `getHeroAnalysis`의 `abillity` 데이터 활용
- 각 스탯별 10개 구간 유저 수 데이터에서 최다 구간을 추출
- 최다 구간을 "추천 세팅 구간"으로 표시 (예: 속도 206~222)
- 히스토그램 차트로 분포 시각화

**유효 옵션 자동 판별 로직**
- 관리자 수동 입력 없이 API 데이터에서 자동 계산
- 판별 기준: 10개 구간 중 최다 구간이 6번째 이상(상위 50%)이면 유효 옵션으로 자동 지정
- 예시:
  - 비브리스 속도: 최다 구간 = 7번째 → 유효 옵션 O
  - 아룬카 속도: 최다 구간 = 1번째 → 유효 옵션 X
  - 아룬카 방어력: 최다 구간 = 10번째 → 유효 옵션 O
- 유효 옵션으로 판별된 스탯은 강조 표시 + 추천 구간 수치 표시

**함께 쓰면 좋은 영웅 / 상대하기 어려운 영웅**
- 공식 API 데이터 top5
- 영웅 아이콘 + 이름 + 비율

**기본 스킬 정보**
- 스킬명, 스킬 설명 (공식 데이터)

#### SEO
- Title: "{영웅명} 장비 세팅 & 스탯 - Epic Sense 에픽세븐 영웅 도감"
- Description: "{영웅명}의 주요 장비 세트, 평균 스탯, 시너지 영웅을 확인하세요."

### 5.4 장비 추천 (/gear-recommend)

#### 목적
유저가 획득한 장비의 스탯을 입력하면 해당 장비를 착용하기 적합한 영웅을 추천해주는 도구.

#### 기능 상세

**장비 필터 (상단)**
- 세트 선택: 전체 세트 목록 (분노/저항/신속/... 등)
- 부위 선택: 무기/투구/갑옷/목걸이/반지/신발
- 주 옵션 선택: 드롭다운

**장비 옵션 입력**
- 부 옵션 4개 입력 필드
  - 옵션 종류 선택 (드롭다운): 공격력%, 방어력%, 생명력%, 속도, 치명확률, 치명피해, 효과적중, 효과저항 등
  - 수치 입력 (숫자 필드)
- 강화 단계 표시 (+0 ~ +15)

**추천 결과**
- 입력한 옵션들이 유효 옵션에 해당하는 영웅 목록
- 매칭 점수 표시 (유효 옵션 몇 개 일치하는지)
- 영웅 아이콘 + 이름 + 직업 + 유효 옵션 표시
- 해당 영웅의 상세 페이지 링크

**영웅별 유효 옵션 데이터**
- 공식 API `getHeroAnalysis` 데이터에서 자동 계산 (관리자 수동 입력 불필요)
- 각 스탯별 10개 구간 중 최다 구간이 6번째 이상이면 유효 옵션으로 자동 판별
- 예: 비브리스 → 자동 판별 결과: ['생명력%', '방어력%', '속도']
- 장비 추천 시 유저가 입력한 옵션과 자동 판별된 유효 옵션을 실시간 매칭

**전체 영웅 추천 / 보유 영웅 추천 탭**
- 로그인 시 보유 영웅 중에서만 추천 가능

- Title: "에픽세븐 장비 추천 - 장비 옵션으로 최적 영웅 찾기 | Epic Sense"

### 5.5 장비 강화 서포터 (/gear-enhance)

#### 목적
장비 강화 시 현재까지 붙은 옵션 수치를 입력하면 계속 강화할 가치가 있는지 판단해주는 도구.

#### 기능 상세

**장비 설정**
- 장비 부위 선택: 무기/투구/갑옷 | 목걸이/반지/신발 (그룹 선택)
- 현재 강화 단계 선택: +0 / +3 / +6 / +9 / +12 / +15
- 초기화 버튼

**옵션 수치 입력 (슬라이더 + 직접 입력)**
- 공격력%
- 방어력%
- 생명력%
- 속도
- 치명확률
- 치명피해
- 효과적중
- 효과저항
- 각 옵션별 슬라이더 (범위: 0 ~ 최대값)
- 수치 직접 입력 가능

**결과 판단**
- "확인하기" 버튼 클릭 시 판단
- 현재 장비 총 점수 표시
- 판단 결과:
  - 🟢 "계속 강화하세요!" — 현재 점수가 기댓값 이상
  - 🟡 "애매합니다. 선택하세요" — 기댓값 근처
  - 🔴 "강화를 중단하세요" — 기댓값 미달
- 각 단계별 기댓값 대비 현재 수치 비교 차트

**판단 로직**
- 강화 단계별 옵션 기댓값은 Supabase DB에 관리자가 입력

## 6. 데이터베이스 스키마 (Supabase)

```sql
-- 유저
users (
  id uuid PRIMARY KEY,
  email text UNIQUE,
  nickname text,
  role text DEFAULT 'user', -- 'user' | 'admin'
  created_at timestamptz
)

-- 보유 영웅 (로그인 유저)
user_heroes (
  id uuid PRIMARY KEY,
  user_id uuid REFERENCES users(id),
  hero_code text,
  created_at timestamptz
)

-- 장비 강화 기댓값 (관리자 입력)
gear_enhance_values (
  id uuid PRIMARY KEY,
  gear_group text, -- 'weapon_group' | 'accessory_group'
  enhance_level int, -- 3, 6, 9, 12, 15
  stat_type text, -- 'atk_pct', 'def_pct', 'hp_pct', 'spd', ...
  expected_value float,
  updated_at timestamptz
)

-- 문의
contacts (
  id uuid PRIMARY KEY,
  name text,
  email text,
  type text,
  content text,
  created_at timestamptz
)

-- 공지/업데이트 내역
notices (
  id uuid PRIMARY KEY,
  title text,
  content text,
  created_at timestamptz
)
```

## 11. 주요 주의사항

- **e7api.onstove.com**은 스마일게이트 비공개 내부 API이므로, 서비스 구조 변경 시 프록시 Worker만 수정하면 대응 가능하도록 설계
- 영웅 이미지는 공식 에픽세븐 CDN URL 사용 (직접 저장 시 저작권 문제 소지)
- 팬 사이트임을 명시하고 상업적 이용이 아님을 푸터에 표시
- Google AdSense 심사를 위해 개인정보처리방침 페이지 필요 (/privacy)
- Cloudflare Pages 무료 플랜 기준: 월 500빌드, 무제한 요청
